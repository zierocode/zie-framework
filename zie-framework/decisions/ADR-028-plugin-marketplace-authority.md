# ADR-028: Plugin Marketplace as Decoupled Authority

Date: 2026-03-30
Status: Accepted

## Context

When zie-memory managed its own marketplace with a hardcoded SHA pin to zie-framework, changes to zie-framework required zie-memory to be updated and released in lockstep. This created a cross-repo dependency where zie-memory could not release independently and broke when zie-framework and zie-memory released on different cadences. Specifically, zie-framework v1.13.0 attempted to release before zie-memory could update its marketplace.json.

## Decision

The Claude Code `settings.json` file (via `extraKnownMarketplaces`) is the **single authority** for plugin discovery. Each plugin (zie-framework, zie-memory) publishes to GitHub marketplace independently. No plugin knows about other plugins. All plugins are fetched from the same `settings.json` marketplace authority.

## Consequences

**Positive:**
- Plugins release independently on their own cadence
- No cross-repo version pinning or dependency graphs
- Changes to zie-framework do not require zie-memory release cycle
- Each plugin controls its own version number and marketplace entry

**Negative:**
- User must manage marketplace entries in settings.json manually (not auto-generated by one plugin)
- Two plugins with incompatible APIs could coexist; no enforcement of compatibility
- Plugins cannot reference each other's versions for validation

**Neutral:**
- Plugins are loaded sequentially; no multi-plugin initialization order dependency
- Settings.json is owned by Claude Code, not by zie-framework
