# Add /zie-fix command — bug path

ตอนนี้เมื่อเจอ bug ต้องใช้ /zie-idea ซึ่ง force ผ่าน brainstorming flow ที่ยาวเกินไปสำหรับ bug fix และไม่มี systematic debugging path

สร้าง /zie-fix ที่ skip ideation เริ่มด้วย systematic debugging ทันที (อ่าน error, reproduce, isolate root cause, fix, verify) integrate กับ zie-memory เพื่อ recall past bugs ใน domain เดียวกัน และ store root cause + fix pattern หลัง resolve
