# Add .coverage to make clean

## Problem

`make clean` removes `__pycache__`, `.pytest_cache`, and `.pyc` files but leaves
`.coverage` behind. The `.coverage` file is a SQLite database generated by
`coverage run` — it grows with each test run and accumulates stale data when
hook files are renamed or deleted.

## Motivation

Consistency: all test artifacts should be cleaned by `make clean`. The `.coverage`
file holding stale paths can cause confusing coverage reports that reference files
that no longer exist.

## Rough Scope

- Add `.coverage` and `coverage.xml` and `htmlcov/` to the `clean` target in Makefile
- Verify `make clean && make test-unit` produces a clean coverage report
- One-line change, no tests needed
